# Baseline 2026-04-21

Базовое окно измерения для capacity-работ по QFL. Все `pg_stat_statements` сравниваются по delta за окно, а не по накопленным totals.

## Как снимать baseline

1. При необходимости очистить накопленную статистику:
   ```sql
   SELECT pg_stat_statements_reset();
   ```
2. Дать системе поработать одинаковое окно наблюдения, например `30-60` минут.
3. Снять метрики до и после окна и сравнивать дельту.

## Метрики окна

- Host load / CPU / RAM:
  ```bash
  ssh debian@kmff.kz 'uptime && docker stats --no-stream --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}" | head -20'
  ```
- `pg_stat_statements` top queries:
  ```bash
  ssh debian@kmff.kz 'docker exec qfl-db psql -U qfl -c "SELECT query, calls, mean_exec_time, total_exec_time FROM pg_stat_statements WHERE query LIKE '\''%player_tour_stats%'\'' OR query LIKE '\''%player_season_stats%'\'' OR query LIKE '\''%goal_rank%'\'' ORDER BY total_exec_time DESC LIMIT 10;"'
  ```
- `seasons` table scans:
  ```bash
  ssh debian@kmff.kz 'docker exec qfl-db psql -U qfl -c "SELECT seq_scan, idx_scan FROM pg_stat_user_tables WHERE relname='\''seasons'\'';"'
  ```
- Goal-video noise:
  ```bash
  ssh debian@kmff.kz 'docker logs qfl-celery-worker --since 1h | grep -c "listed=0 matched=0"'
  ```
- SSR TTFB:
  ```bash
  curl -w "%{time_starttransfer}\n" -o /dev/null -s https://kffleague.kz/ru/matches/932
  ```

## Expected delta после PR1-PR3

- `seasons.seq_scan` падает кратно после cache-first.
- `listed=0 matched=0` падает примерно в `2.5x` после `GOAL_VIDEO_SYNC_INTERVAL_MINUTES=5`.
- Для extended stats появляется однозначная классификация bottleneck:
  - `DB-dominant`
  - `Sleep-dominant`
  - `RTT-dominant`
  - `Mixed`

## Goal-rank operational checklist

Перед любым refactor `goal_rank` снять:

```bash
ssh debian@kmff.kz 'docker exec qfl-db psql -U qfl -c "SELECT count(*) FROM player_season_stats;"'
ssh debian@kmff.kz 'docker exec qfl-db psql -U qfl -c "SELECT pid, state, wait_event_type, wait_event, query FROM pg_stat_activity WHERE datname = current_database() AND query ILIKE '\''%player_season_stats%'\'';"'
ssh debian@kmff.kz 'docker exec qfl-db psql -U qfl -c "SELECT relname, n_dead_tup, last_autovacuum, last_autoanalyze FROM pg_stat_user_tables WHERE relname = '\''player_season_stats'\'';"'
```

## Deployment checklist

- Обновить `/home/debian/qfl/.env` на primary host:
  - `WEB_STATEMENT_TIMEOUT_MS=30000`
  - `SOTA_DEAD_SEASON_MIN_404=30`
  - `SOTA_DEAD_SEASON_404_RATIO=0.8`
  - `SOTA_DEAD_SEASON_TTL_SECONDS=3600`
  - `DEBUG_SYNC_TIMINGS=false`
  - `GOAL_VIDEO_SYNC_INTERVAL_MINUTES=5`
- Для media-host синхронизировать `.env.media` с `GOAL_VIDEO_SYNC_INTERVAL_MINUTES=5`.
- Включить server-side docker log rotation:
  - `driver: json-file`
  - `max-size: 10m`
  - `max-file: 3`
