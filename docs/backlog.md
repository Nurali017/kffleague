**Контекст:** SOTA API `/api/public/v1/metrics/` возвращает 293 метрики (id, key, name). Сейчас не используем.

## [SOTA-5] Критические изменения API — сезон 2026

**Приоритет:** CRITICAL
**Дата обнаружения:** 2026-03-06
**Контекст:** SOTA API изменил структуру ответов. Текущий sync код сломан в нескольких местах.

### 5a. Поле `start_date` / `end_date` в seasons

`reference_sync.py:94` использует `s_ru.get("date_start")`, но API теперь возвращает `start_date` / `end_date`.

**Было:** `{"date_start": "2025-03-01", "date_end": "2025-10-27"}`
**Стало:** `{"start_date": "2025-03-01", "end_date": "2025-10-27"}`

### 5b. Seasons — имена теперь только год

API возвращает `"name": "2026"` вместо `"Казахстанская Премьер-лига 2025"`. `Accept-Language` не влияет. Все 3 языковых запроса возвращают одинаковое `"2026"`.

**Решение:** не перезаписывать `name`/`name_kz`/`name_en` из SOTA, оставить ручное управление через админку.

### 5c. Teams — потеряны logo, city, short_name

Эндпоинт `/teams/` теперь возвращает только `{id, name, tournaments, seasons}`. Пропали: `logo`, `short_name`, `country`, `stadium`, `city`.

**Impact:** `sync_teams()` будет записывать `None` в `logo_url` и `city`.
**Решение:** убрать `logo_url` и `city` из SOTA sync, оставить только `name` по 3 языкам. Логотипы берём из MinIO (уже работает через `sync_team_logos`).

### 5d. `pass_accuracy` пропал из game stats v1

`/games/{id}/players/` и `/games/{id}/teams/` больше не возвращают `pass_accuracy`. Было 12 ключей, стало 11.

### 5e. Players — `team` → `teams` (массив)

Было: `"team": {"id": 94, "name": "Кайсар"}`
Стало: `"teams": [94]`

**Что сделать:**
- [ ] Не синкать season dates из SOTA — `date_start` / `date_end` ведём локально
- [ ] Не перезаписывать season names из SOTA
- [ ] Убрать `logo_url`, `city` из teams sync (оставить только names)
- [ ] Подтвердить, что player sync игнорирует `team` / `teams`
- [ ] Оставить graceful fallback для `pass_accuracy`
- [ ] Добавить local season_id=200 в `SYNC_SEASON_IDS` (`sota_season_id=173`)
