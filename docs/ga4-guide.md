# GA4 — Практический гайд для kffleague.kz

## 1. Быстрый старт — проверка что всё работает

1. Откройте `kffleague.kz/team/1` в браузере (без блокировщика рекламы)
2. В другой вкладке откройте GA4 → **Отчёты → В реальном времени**
3. В течение ~30 секунд в блоке **События** должно появиться событие `team_view`

### Если `team_view` не появился

- **Блокировщик рекламы** — uBlock, AdBlock и другие блокируют GA4. Проверьте в режиме инкогнито без расширений
- **Переменная окружения** — убедитесь что в `.env.local` (или в Docker) задана:
  ```
  NEXT_PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXXXXX
  ```
- **Неверный Measurement ID** — проверьте ID в GA4 → Администратор → Потоки данных → ваш стрим → Идентификатор потока данных
- **Кеш/сборка** — после изменения переменных нужно пересобрать и перезапустить фронтенд

## 2. Обязательная настройка: Custom Dimensions

> **Без этого шага параметры событий (`team_name`, `game_id` и др.) не появятся в отчётах.** Нужно сделать один раз после подключения GA4.

**Путь навигации:**
`GA4 → Администратор (шестерёнка внизу слева) → Ресурс → Специальные определения → Специальные параметры → кнопка "Создать специальный параметр"`

Создайте 5 параметров с областью действия **Событие**:

| Название параметра | Параметр события | Зачем нужно |
|---|---|---|
| `team_id` | `team_id` | Фильтровать события по конкретной команде |
| `team_name` | `team_name` | Читаемые названия команд в отчётах |
| `game_id` | `game_id` | Фильтровать события по матчу |
| `is_live` | `is_live` | Отделить просмотры live-матчей от записей |
| `tab_name` | `tab_name` | Видеть какие вкладки открывают на странице команды |

**Как заполнять форму:**
- **Название параметра** — то что вы видите в отчётах (можно любое название)
- **Область действия** → выбрать `Событие`
- **Параметр события** — точное название параметра из таблицы выше (строчными буквами, как написано)
- Нажать **Сохранить**

Повторить для каждого из 5 параметров. После создания данные начнут появляться в отчётах (к старым данным не применится).

## 3. Все события — что они означают и где смотреть

Путь к любому событию: `Отчёты → Взаимодействие → События → нажать на название события`

### `team_view`
**Что происходит:** пользователь открыл страницу команды
**Параметры:** `team_id`, `team_name`
**Зачем:** рейтинг популярности команд

### `player_view`
**Что происходит:** пользователь открыл страницу игрока
**Параметры:** `player_id`, `player_name`, `team_id`, `team_name`
**Зачем:** самые просматриваемые игроки

### `match_view`
**Что происходит:** пользователь открыл страницу матча
**Параметры:** `game_id`, `is_live` (`true`/`false`)
**Зачем:** просмотры матчей, отдельно live и обычных

### `news_read`
**Что происходит:** пользователь открыл новость
**Параметры:** `news_id`, `news_title`
**Зачем:** самые читаемые материалы

### `ticket_click`
**Что происходит:** пользователь нажал кнопку покупки билета
**Параметры:** `game_id`
**Зачем:** конверсия в покупку (самое важное коммерческое событие)

### `live_banner_click`
**Что происходит:** пользователь нажал на live-баннер
**Параметры:** `game_id`
**Зачем:** эффективность баннера прямых трансляций

### `youtube_stream_click`
**Что происходит:** пользователь нажал на ссылку YouTube-трансляции
**Параметры:** `game_id`
**Зачем:** сколько людей переходит смотреть матч на YouTube

### `stats_tab_click`
**Что происходит:** пользователь переключил вкладку на странице команды
**Параметры:** `team_id`, `team_name`, `tab_name`
**Зачем:** какие разделы (состав, расписание, статистика и т.д.) интересны

## 4. Готовые ответы на бизнес-вопросы

### "Какую команду смотрят больше всего?"
1. `Отчёты → Взаимодействие → События`
2. Нажать на `team_view`
3. В таблице внизу изменить основной параметр на `team_name` (кликнуть на заголовок столбца → выбрать Специальный параметр → `team_name`)
4. Сортировать по **Количество событий** — по убыванию

### "Сколько человек кликнули на билеты за неделю?"
1. `Отчёты → Взаимодействие → События`
2. Нажать на `ticket_click`
3. В верхней части установить диапазон дат: последние 7 дней
4. Смотреть **Количество событий** в сводке сверху

### "Кто сейчас смотрит матч #42?"
1. `Отчёты → В реальном времени`
2. В блоке **Просмотры страниц** — найти `/matches/42`
3. Или нажать **Просмотр снимка пользователя** → фильтр `Путь страницы` → содержит `/matches/42`

### "Что популярнее: матчи или новости?"
1. `Отчёты → Взаимодействие → События`
2. В таблице найти строки `match_view` и `news_read`
3. Сравнить столбец **Количество событий** за нужный период

### "Какие вкладки команды никто не смотрит?"
1. `Исследования → + (создать новое исследование) → Пустой`
2. В **Параметры** добавить: `tab_name` (Специальный параметр), `Название события`
3. В **Показатели** добавить: `Количество событий`
4. Перетащить `Название события` в **Фильтры** → `точно соответствует` → `stats_tab_click`
5. Перетащить `tab_name` в **Строки**, `Количество событий` в **Значения**
6. Видно какие вкладки не кликают (0 или минимальные значения)

### "Откуда приходят пользователи?"
1. `Отчёты → Источники трафика → Привлечение трафика`
2. Таблица покажет: Organic Search, Direct, Social и т.д.
3. Для детализации нажать на источник или изменить параметр на `Источник/канал сеанса`

### "На каком устройстве смотрят live?"
1. `Исследования → Пустое исследование`
2. Параметры: `Категория устройства`
3. Показатели: `Количество событий`
4. Фильтр: `Название события` точно соответствует `match_view` **И** `is_live` точно соответствует `true`
5. Строки: `Категория устройства` → видно desktop/mobile/tablet

## 5. Кастомный отчёт "Рейтинг команд"

Пошаговая инструкция создания исследования:

1. `Исследования` (в левом меню) → `+ Пустое исследование`
2. Дать название: `Рейтинг команд`
3. В блоке **Переменные** (слева):
   - Нажать `+` рядом с **Параметры** → найти и добавить: `team_name`, `Название события`
   - Нажать `+` рядом с **Показатели** → найти и добавить: `Количество событий`
4. В блоке **Настройки** (в центре):
   - Перетащить `team_name` в **Строки**
   - Перетащить `Количество событий` в **Значения**
   - В **Фильтры** нажать `+` → `Название события` → `точно соответствует` → `team_view`
5. Нажать на заголовок столбца `Количество событий` → сортировка по убыванию
6. Готово — список команд по убыванию просмотров

Чтобы сохранить: отчёт сохраняется автоматически. Открыть позже: `Исследования → ваш отчёт в списке`.

## 6. Realtime мониторинг во время матча

Удобно держать на втором мониторе во время прямой трансляции:

1. Открыть `GA4 → Отчёты → В реальном времени`
2. В блоке **Просмотры страниц** смотреть строки с `/matches/` — там количество активных пользователей
3. Для конкретного матча:
   - Нажать **Просмотр снимка пользователя** (кнопка в правом верхнем углу блока «Пользователи»)
   - Добавить фильтр: `Путь страницы` → `содержит` → `/matches/42` (нужный ID матча)
4. Цифра «Пользователи за последние 30 мин» в шапке — общее количество на сайте прямо сейчас

**Что видно в режиме реального времени:**
- Сколько человек на каком матче прямо сейчас
- Из каких стран и городов
- С каких устройств (mobile/desktop)
- Какие события происходят (клики на билеты, YouTube и т.д.)

## 7. Настройка конверсии для ticket_click

Чтобы `ticket_click` отображался в разделе «Конверсии» (важно для отчётности):

1. `GA4 → Администратор (шестерёнка) → Ресурс → События`
2. В списке найти `ticket_click`
3. Переключить тумблер в столбце **Отметить как конверсию** в положение ВКЛ
4. Подождать 24-48 часов — событие появится в `Отчёты → Монетизация → Конверсии`

После этого в сводке `Отчёты → Реклама → Пути конверсии` будет видно весь путь пользователя до клика на билет (откуда пришёл → что смотрел → нажал на билет).
